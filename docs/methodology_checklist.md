# 論文再現→実務検証プロジェクト 方法論チェックリスト

PCA_SUBプロジェクト（8フェーズ、約3000行、23テスト）で確立した手順。
次のプロジェクトで「最初から効率良く、同じ失敗を繰り返さない」ためのリスト。

---

## Phase 1: 論文再現

- [ ] 論文を通読し、数式・アルゴリズム・データを正確に理解
- [ ] 対象銘柄・データ期間・パラメータデフォルトを constants.py で一元管理
- [ ] データ取得は provider abstraction（yfinance/CSV切替可能）
- [ ] **カレンダー整列**（日米の祝日・金→月対応など）を専用モジュールで実装
- [ ] リターン定義（CC, OC等）を明確に定義し、モジュール化
- [ ] 論文の全ベースライン戦略を実装（比較対象がないと判断できない）
- [ ] 最低限のテスト（直交性、対称性、次元、未来情報なし）
- [ ] **最初のバックテスト結果を出す**（Grossで論文と定性一致するか確認）

### よくある罠
- データがない期間の銘柄（XLC等）でNaNが発生 → pairwise相関で対処
- yfinanceのauto_adjust価格と論文データの差異 → 定量的一致ではなく定性的一致を目指す

---

## Phase 2: コスト分析（ここを飛ばさない）

- [ ] 取引コストモデルを実装（片道コスト × ターンオーバー）
- [ ] Breakeven分析: 「何bpまでなら黒字か」を計算
- [ ] 対象ETF/銘柄の**実際の売買代金**を確認（想像より薄い場合がほとんど）
- [ ] Gross → Net の落差を数字で把握
- [ ] **この時点で「コスト的に致命的か、勝負になるか」を判断**

### 学んだこと
- 日中L/S戦略（毎日全建替）は片道5bpで年間50%超のコストが発生する
- TOPIX-17 ETFは大半が日次売買代金1000万〜3000万円 → 機関投資家には不足
- Breakeven分析は「この戦略を続ける意味があるか」の最初の分岐点

---

## Phase 3: 感度分析

- [ ] 主要パラメータを1つずつ変えて成績変化を確認
- [ ] 目的は「最適化」ではなく「ロバスト性の確認」
- [ ] ロバストなパラメータ（変えても変わらない）は以降いじらない
- [ ] 敏感なパラメータ（大きく変わる）はOOS検証の対象にする

### 学んだこと
- 推定ウィンドウ長・正則化強度・相関行列期間はロバストだった
- 主成分数Kは最も敏感 → OOS検証が必要
- リバランス頻度は3日以上で急激に劣化 → シグナルの半減期が分かる

---

## Phase 4: ファクター分解・α帰属分析

- [ ] モデルの各成分を個別に取引した場合の成績を計測
- [ ] **αの何%がどの成分から来ているか**を定量化
- [ ] 伝播構造（何→何がリード/ラグしているか）を可視化

### 学んだこと
- PCA_SUBではαの95%が1つのファクターから来ていた
- これが判明してから、改善実験の焦点が完全に定まった
- **早めにやるべき**（感度分析と同時期が良い）

---

## Phase 5: OOS検証

- [ ] Train / Validation / Test を厳密に分離
- [ ] すべての閾値・パラメータ選択はTrain内のみで決定
- [ ] Walk-forward（年単位のローリング選択+評価）を実施
- [ ] **Test期間の数字だけで最終判断する**

### 学んだこと
- in-sampleで効いたK=4-5がwalk-forwardで逆効果になった
- 条件付き取引はOOSでも黒字を維持した（ロバスト）
- Full期間の数字は参考値。Testの数字が判断基準。

---

## Phase 6-7: 実務化設計と改善実験

- [ ] 執行対象の改善（ETF → 個別株バスケットなど）
- [ ] 条件付き取引（強い日だけ打つ）
- [ ] ポートフォリオ集中（分位点を絞る）
- [ ] ウェイト設計（等ウェイト vs スコア比例 vs rank比例）

### 効いたもの
1. **条件付き取引**（最大の改善レバー）
2. **個別株バスケット化**（流動性・コスト改善）
3. **集中投資 q=0.2**（Net AR +1.6pp）

### 効かなかったもの
1. Walk-forward K（ノイズ増で悪化）
2. 複合フィルタ（条件間の相関が高く冗長）
3. スコア比例ウェイト（スコア差が小さく効果なし）
4. セクター限定（分散低下のデメリットが上回る）

---

## Phase 8: 最終判断

- [ ] 事前に合意した数値基準で判定
- [ ] 判定は「メイン戦略 / サブ戦略 / 不採用」の3択
- [ ] 試してダメだったことも含めて最終レポートを書く
- [ ] 投資家向けの平易な総括テキストを作成

## やってはいけないこと（失敗から学んだ）

1. **Gross成績だけ見て「強い」と判断する** → コストを入れると半分以上の戦略が赤字になる
2. **Phase番号を途中で別系列にする** → 混乱する。findingsと一貫させる
3. **パラメータ最適化をOOSなしでやる** → in-sampleの「最適」はOOSで裏目に出る
4. **全日取引前提でコスト計算する** → 条件付き取引で劇的に変わるので、先に取引日フィルタを考える
5. **流動性を確認せずにバックテストする** → ETFの売買代金が想定の1/10だった経験あり
6. **複雑な改善を大量に試す** → 2-3個に絞って見切る方が効率的
